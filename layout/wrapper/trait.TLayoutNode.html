Allows some convenience methods on generic layout nodes.

Required Methods

unsafe fn new_with_this_lifetime(&self, node: &LayoutJS<Node>) -> Self

Creates a new layout node with the same lifetime as this layout node.

fn type_id(&self) -> Option<NodeTypeId>

Returns the type ID of this node. Fails if this node is borrowed mutably. Returns None if this is a pseudo-element; otherwise, returns Some.

unsafe fn get_jsmanaged<'a>(&'a self) -> &'a LayoutJS<Node>

Returns the interior of this node as a LayoutJS. This is highly unsafe for layout to call and as such is marked unsafe.

fn text_content(&self) -> Vec<ContentItem>

If this is a text node or generated content, copies out its content. If this is not a text node, fails.

FIXME(pcwalton): This might have too much copying and/or allocation. Profile this.

fn first_child(&self) -> Option<Self>

Returns the first child of this node.
