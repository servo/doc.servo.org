pub unsafe fn pipe() -> IoResult<Pipe>

Creates a new low-level OS in-memory pipe.

This function can fail to succeed if there are no more resources available to allocate a pipe.

This function is also unsafe as there is no destructor associated with the Pipe structure will return. If it is not arranged for the returned file descriptors to be closed, the file descriptors will leak. For safe handling of this scenario, use std::old_io::PipeStream instead.
