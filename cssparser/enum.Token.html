One of the pieces the CSS input is broken into.

Some components use CowString in order to borrow from the original input string and avoid allocating/copying when possible.

Variants

Ident

A <ident-token>

AtKeyword

A <at-keyword-token>

The value does not include the @ marker.

Hash

A <hash-token> with the type flag set to "unrestricted"

The value does not include the # marker.

IDHash

A <hash-token> with the type flag set to "id"

The value does not include the # marker.

QuotedString

A <string-token>

The value does not include the quotes.

Url

A <url-token> or url( <string-token> ) function

The value does not include the url( ) markers or the quotes.

Delim

A <delim-token>

Number

A <number-token>

Percentage

A <percentage-token>

Dimension

A <dimension-token>

UnicodeRange

A <unicode-range-token>

Components are the start and end code points, respectively.

The tokenizer only reads up to 6 hex digit (up to 0xFF_FFFF), but does not check that code points are within the range of Unicode (up to U+10_FFFF).

WhiteSpace

A <whitespace-token>

Comment

A comment.

The CSS Syntax spec does not generate tokens for comments, But we do, because we can (borrowed &str makes it cheap).

The value does not include the /* */ markers.

Colon

A : <colon-token>

Semicolon

A ; <semicolon-token>

Comma

A , <comma-token>

IncludeMatch

A ~= <include-match-token>

DashMatch

A |= <dash-match-token>

PrefixMatch

A ^= <prefix-match-token>

SuffixMatch

A $= <suffix-match-token>

SubstringMatch

A *= <substring-match-token>

Column

A || <column-token>

CDO

A <!-- <CDO-token>

CDC

A --> <CDC-token>

Function

A <function-token>

The value (name) does not include the ( marker.

BadUrl

A <bad-url-token>

This token always indicates a parse error.

BadString

A <bad-string-token>

This token always indicates a parse error.

CloseParenthesis

A <)-token>

When obtained from one of the Parser::next* methods, this token is always unmatched and indicates a parse error.

CloseSquareBracket

A <]-token>

When obtained from one of the Parser::next* methods, this token is always unmatched and indicates a parse error.

CloseCurlyBracket

A <}-token>

When obtained from one of the Parser::next* methods, this token is always unmatched and indicates a parse error.
