Unsigned multi-precision integer arithmetic.

Limbs ordered least-significant-limb to most-significant-limb. The bits limbs use the native endianness.

  • fold_5_bit_windows
    Processes limbs as a sequence of 5-bit windows, folding the windows from most significant to least significant and returning the accumulated result. The first window will be mapped by init to produce the initial value for the accumulator. Then f will be called to fold the accumulator and the next window until all windows are processed. When the input’s bit length isn’t divisible by 5, the window passed to init will be partial; all windows passed to fold will be full.
