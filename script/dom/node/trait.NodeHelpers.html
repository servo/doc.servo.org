fn dirty(self, damage: NodeDamage)

Marks the given node as IS_DIRTY, its siblings as HAS_DIRTY_SIBLINGS (to deal with sibling selectors), its ancestors as HAS_DIRTY_DESCENDANTS, and its descendants as IS_DIRTY. If anything more than the node's style was damaged, this method also sets the HAS_CHANGED flag.

fn force_dirty_ancestors(self, damage: NodeDamage)

Similar to dirty, but will always walk the ancestors to mark them dirty, too. This is useful when a node is reparented. The node will frequently already be marked as changed to skip double-dirties, but the ancestors still need to be marked as HAS_DIRTY_DESCENDANTS.

See #4170
